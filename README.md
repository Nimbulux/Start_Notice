# Start_Notice

一个防止重复启动与补全启动提示的小工具

> [!IMPORTANT]
> 请注意 本项目由AI编写

## 使用方法
1. 在与 `Start_Notice.exe` 相同的目录下创建 `config.txt`，文件编码必须为 **UTF‑8**。
2. 在 `config.txt` 中写入需要启动的目标程序相对路径，每行一个，例如：
   ```text
   Hello_World.exe
   ```

> [!NOTE]
> - **只能使用相对路径**。程序会自动将自身所在目录作为基准，与文件内写入的路径拼接，得到完整的启动路径。
> - 目标程序也必须放置在此目录下（或其子目录中，可使用类似 `subdir/app.exe` 的相对路径）。
> - 路径拼接示例：  
>   本程序位于 `C:\User\sample\Desktop\Start_Notice.exe`，  
>   配置文件中填写 `Hello_World.exe`，  
>   拼接后的实际启动路径即为 `C:\User\sample\Desktop\Hello_World.exe`。

## 启动画面
为了提供清晰的反馈，降低等待时的焦躁感，程序加入了启动画面。

### **如果有 `splash.png`**  
  若目录下存在名为 `splash.png` 的图片文件，程序会将该图片无边框居中渲染至屏幕中央。  
  > [!NOTE]  
  > 图片的像素尺寸即实际屏幕上的渲染大小，请留意控制分辨率。

### **如果没有 `splash.png`**  
  当图片不存在时，程序会显示一个默认的 **400×400** 像素窗口：背景纯白、无标题栏，文字 **“正在启动，请稍候…”** 居中显示。

启动画面会一直保持显示，直到 `config.txt` 中所指定的程序出现窗口后自动隐藏。

## 背景
一个小小的工具。因为学校的老师不太会用电脑，电脑响应慢时容易多次点击，造成程序重复启动，甚至白板软件会出现一个透明的覆盖层挡在屏幕上。为了彻底解决这个问题，于是写了这个启动提示工具，确保程序只启动一次，并提供友好的等待提示。

## 注意事项
> [!IMPORTANT]
> 本程序没有图形界面，所有配置均需通过手动编辑 `config.txt` 完成。
> - `config.txt` 必须与 `Start_Notice.exe` 放在同一文件夹中。
> - 目标程序也需要放置在该目录下（或其子目录中）。
> - 推荐将本程序直接拷贝到目标软件的目录下使用。
> - 部分白板软件带有自动更新功能，快捷方式所指向的程序可能无法稳定运行，此时建议将 `Start_Notice.exe`（连同 `config.txt`）直接复制到白板的实际版本文件夹中再使用。
